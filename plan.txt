Arquitetuta de repositório:
  - Um único repositório
  - Uma pasta para cada serviço representando um microsserviço
  - Em cada microsserviço terá uma divisão básica em camadas
    - models
    - services
    - schemas
    - core
    - api
    - db
  - Cada microsserviço terá suas dependências, Dockerfile, .env e .ignores


Serviços:
  - auth e users
  - payments e notification
  - events
  - orders


Tecnologias:
  - FastAPI
  - Pydantic
  - SQLAlchemy
  - PostgreSQL
  - Poetry
  - Redis


Padrões git:
  - duas branchs principais main e dev
  - Uma branch auxiliar para mergear na dev
  - usaremos o padrão de nomenclatura das branchs sendo feat/nome-do-feature e fix/nome-do-bug
  - os Pull Requests devem ser feitos na branch dev, para só no fim do primeiro MVP ir para a main
